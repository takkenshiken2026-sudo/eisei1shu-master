## 優先度順の次アクション

### 優先度 高 — meta description の主題語フロントロード（実装可・検証可）
タイトルは主題語を前置できたが、**meta description は依然として汎用の接頭辞が先頭**にある。
- 現状（一問一答の例）:
  `第一種衛生管理者試験の一問一答 令和2年_4月_問36_選択肢2・労働生理。皮膚感覚には…`
- 課題: 検索スニペットで最初に見える語が「一問一答 令和2年_4月_問36_選択肢2・労働生理」という
  識別子であり、ユーザーの検索語（＝設問内容）と噛み合わない。
- 改善案: `tools/q_page_seo.py` の `question_meta_description()` を、設問内容（body）を先頭に置く構成へ。
  タイトル改善と同じ考え方で、過去問・実践演習・一問一答の全問ページに適用。
- 影響/検証: 2,066+ ページ。CI の内部リンク・サイト統合検証で回帰確認が可能。

### 優先度 高 — 検索意図一致（クエリとタイトル/見出しの語ズレ）
- 例: クエリ「産業医 勧告」に対しページタイトルは「産業医の権限とは？」。
  「勧告（勧告権）」は 産業医 の具体的権限であり、**タイトル・H2・FAQ に「勧告」を明示**すると一致度が上がる。
- 対象は GSC で page 4〜5 に出ている用語ページを個別点検（勧告／受験料／合格後 等）。

### 優先度 中 — page 4 の受け皿ページへの内部リンク強化
- `衛生管理者 過去問`（最多表示クエリ）の受け皿 `/q/` はトップからリンク済みだが、
  記事群・用語ページ本文からの文脈リンク（アンカーテキスト「過去問」）を増やし、
  ハブへの内部リンク集中度を高める。
- 用語ページ（産業医の権限・産業保健スタッフ 等）へも関連用語・関連記事からの相互リンクを追加。

### 優先度 中 — PC の平均順位低下（PC 30.7 / モバイル 12.6）の要因調査
- viewport・レスポンシブは設定済み（`width=device-width`）で技術的欠陥は見当たらない。
- PC 表示 520 回に対しモバイル 338 回。PC はより広い head クエリ（新規サイトが不利）で
  露出している可能性が高く、クエリ構成の差と推定。**当面は監視**とし、
  Core Web Vitals（LCP/CLS）の実測を次回データで確認する。

### 優先度 低 — 掲載順位の底上げ（サイト権威性）
- 3か月・表示861・クリック10 の新規サイト。上位化には被リンク獲得と継続的な内部リンク整備、
  更新頻度の維持が中心。単一のコード修正で解決する領域ではないため、施策として継続。

## 次に着手する場合の推奨順
1. meta description フロントロード（コード修正・即効・全問ページ）
2. page 4 用語ページの「勧告」等クエリ語の明示（タイトル/見出し/FAQ）
3. ハブ・用語ページへの内部リンク強化
