# GSC 改善レポート（2026-07 / 過去3か月）

Google Search Console のパフォーマンスデータ（検索タイプ: ウェブ / 過去3か月）を分析し、
検索での見え方（CTR）を改善するための修正を行った記録。

## データ概要

| 指標 | 値 |
| --- | --- |
| クリック数（合計） | 10 |
| 表示回数（合計） | 861 |
| 平均 CTR | 約 1.2% |
| 平均掲載順位 | 約 23 位 |

- 国別: ほぼ日本（10 クリック / 827 表示）。
- デバイス別: モバイル 7 クリック（順位 12.6）、PC 3 クリック（順位 30.7）。

## 主要な所見

1. **上位表示されているのにクリックが 0 のページが多い。**
   掲載順位 12 位以内・表示 4 回以上でクリック 0 のページが 30 件以上。
   インデックス・ランキングは機能しているが、**検索結果でのクリック（CTR）が取れていない**のが最大の課題。

2. **過去問（`/q/past/*`）ページのタイトルが、検索クエリと一致していなかった。**
   旧タイトルは全 616 ページが
   `第一種衛生管理者試験 過去問 令和1年10月 第1問（分野）｜一衛マスター`
   のように**同一プレフィックスで始まり、設問の主題（キーワード）を一切含んでいなかった**。
   一方で実際の検索クエリは「成人の呼吸運動で正しいのはどれか」「サルモネラ菌食中毒」「roc 曲線」など、
   **設問文そのもの＝主題語**であり、タイトルと噛み合っていなかった。

3. **年度表記のデータ不具合。**
   `data/past_questions.csv` の `exam_wareki` 列が全行 `令和1年10月月` のように **「月」が重複**しており、
   タイトル・H1・パンくず・構造化データ（全 616 ページ）に誤表記が出ていた。

## 実施した修正

### 1. 過去問タイトルへ「主題語」を前置（CTR 改善）
`tools/q_page_seo.py` に `stem_topic()` と `past_question_page_title()` を追加。
設問文（stem）先頭の主題語を抽出し、タイトル冒頭に置くことで検索クエリとの一致度を高めた。

- 旧: `第一種衛生管理者試験 過去問 令和6年10月 第33問（労働衛生（有害業務））｜一衛マスター`
- 新: `食中毒｜第一種衛生管理者試験 過去問 令和6年10月 第33問｜一衛マスター`

抽出は「◯◯に関する／に定める／について …」等の定型表現を境界にし、「◯◯における△△」型は後段の主題語を優先、
括弧の途中切れや冗長な長さも抑制。生成後の全タイトルが重複なし（ユニーク）で、空タイトル・壊れた括弧がないことを確認済み。

同じ仕組みを**一問一答（`/q/ichimon/*` 950 ページ）・実践演習（`/q/practice/*` 500 ページ）**にも適用。
- 一問一答（例: 検索クエリ「皮膚感覚には、触圧覚、痛覚…」で表示）: `皮膚感覚には、触圧覚、痛覚、温度感覚…｜第一種衛生管理者試験 一問一答 …`
- 実践演習: `SDS｜第一種衛生管理者試験 実践演習 第392問（関係法令（有害業務））｜一衛マスター`
- 過去問 616・実践演習 500・一問一答 950 の計 **2,066 ページすべてがユニークタイトル**であることを検証済み。

### 2. 年度表記の「月月」重複を修正（データ品質）
`data/past_questions.csv` の `exam_wareki` を `令和1年10月月` → `令和1年10月` に一括修正（616 行）。
これによりタイトル・H1・パンくず・JSON-LD の年度表記が正常化。

## 影響範囲と検証

  （`/q/` 以下の HTML はビルド生成物のため CI で再生成される。）
